# セットアップ用スクリーンショット置き場

このフォルダは `setup.html`（セットアップウィザード）が参照するスクリーンショット画像の保管場所です。
各ステップの操作画面を画像で示すことで、初見の導入者が迷わず設定を進められるようにします。

`setup.html` は `images/setup/<ファイル名>` を相対参照します。本フォルダ直下に正しいファイル名で画像を配置してください。

---

## 撮影方針

- **ブラウザ幅を固定**: 撮影時はウィンドウ幅を一定（例: 1280px）に揃え、画像間で見た目のばらつきを抑える。
- **該当箇所だけトリミング**: 操作対象のUIだけを切り出し、余白・無関係なツールバーは含めない。
- **赤枠で対象を強調**: クリック先・入力欄など注目箇所を赤い矩形枠で囲む。線の太さ・色はマニフェスト全体で統一。
- **テーマ統一**: ライト/ダークなどのテーマを全スクショで揃える。混在させない。

---

## 機密マスキング（最重要）

このリポジトリ／`setup.html` は**公開配布される前提**です。画像に機密情報を残さないこと。

- **Bot Token 等の秘密値は単色で完全に塗りつぶす**。ぼかし（モザイク）は復元され得るため、Token のような決定的な秘密値には使わず、**不透明な単色矩形で全塗り**する。
- **テスト用アプリ／ダミーサーバーで撮るのが最も安全**。本番の値が写り込まないよう、可能な限り本番ではなくテスト環境で撮影する。
- **撮影前に画面全体を点検**する:
  - ブックマークバー（個人情報・社内URLが見えがち）
  - メール表示・アカウント名・通知トースト
  - 開いている他のタブのタイトル
  - これらは事前に非表示化／クローズしてから撮る。

---

## 画像フォーマット・サイズ

- **PNG 推奨**（テキストやUIの輪郭が鮮明）。
- **1枚あたり 200KB を目安**に圧縮する。超える場合はトリミング縮小・減色・最適化（pngquant 等）で削減。

## 注記

実画像が未配置の間、`setup.html` では各枠が「（スクリーンショット準備中）」表示になる（`onerror` フォールバック）。
正しいファイル名で画像を配置すれば、自動的に実画像へ差し替わる。
